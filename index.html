CHAPTER 01

My Projects

Need to Go: Again!

ROLE — Lead Writer / Lead Programmer

Need to Go: Again! is a lighthearted, comedic branching narrative game that explores a broke college student desperately searching for a public restroom. Built using Unity + Yarn Spinner, this project was a design exercise in narrative choices and branching paths.

The hardest part of this project wasn't writing the jokes, it was making sure every choice felt like it mattered. Early drafts let players "win" no matter what they picked, which flattened the whole point of a branching narrative. I rebuilt the dialogue graph around three real failure states and mapped every node by hand in Yarn Spinner before writing a single line of final dialogue, which made it much easier to spot dead-end branches and dialogue that contradicted earlier choices.

Gun With It!

ROLE — Systems Design

Gun With It! is a fast-paced, simultaneous combat card game set in a gritty, post-apocalyptic wasteland controlled by the tyrannical El Feo. Players must strategize between using item cards or weapon cards in order to be the last one standing.

As this is a tabletop game, I learned a lot in the process of making this game. Index cards work great for fast iteration and prototyping, but aren't exactly pretty to look at. I had to learn the physical card-crafting workflow for this and am very proud of the end result (Though I had to print several re-tries)

The index-cards went through many many rounds before a single card got real art. The "simultaneous reveal" mechanic was the riskiest part of the design, but it turned out to be the most engaging. During playtests, players often counted down in unison, which really made the game feel gladitorial. However, because all cards are revealed at the same time, this meant players spent a lot of time parsing what just happened. To address this, we ended up color-coding the card backs by category (item vs. weapon) so players could read the game-state at a glance, which is now a trick I will reach for in future tabletop prototypes.

Alien Defense!

ROLE — Lead Programmer / Lead Systems Design

My earliest project, Alien Defense! is a 2d top down tower defense game built in Unity. This is the first ever project I made in a game engine, which I learned a lot from. Although it is a demo, it is fairly feature complete, including 4 distinct towers to choose from (each their own upgrade trees), an in-game economy, 4 distinct enemy types (including a final boss fight) and even stat tracking. You can play the demo HERE:

When I started this project, even navigating the game engine's UI was difficult. However, I was able to use various online resources such as Youtube guides and Unity's documentation to help me. As I said earlier, I definitely learned a lot, but one thing I quickly grasped was the need for version control. So often I would write up a script, test it, only for it to somehow break everything and end spending multiple hours backtracking and tracing what went wrong. This is when I learned the value of github. Version control gave me peace of mind that I could experiment to my heart's content while not needing to stress over every bug. The repo is public, check it out here!
